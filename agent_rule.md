# agent_rule

1. Keep going without interruption.
2. Ignore history. Check the repo for safety issues.
3. If multiple privileged steps are needed, combine them into a single script and run them together. Before execution, briefly summarize what it will do. Include brief usage notes.
4. Do not put repo-related or GitHub-operation scripts in the repo. Put other scripts at the repo `scripts/` and record them in README.md.
5. Do not print full file contents unless I explicitly ask. - Prefer short summaries over verbose output. For code changes, summarize what changed before showing any code.
